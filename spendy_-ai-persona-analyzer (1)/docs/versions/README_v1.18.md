
# Spendy: AI 페르소나 분석기 - 버전 1.18 요구사항 정의서

이 문서는 'Spendy' 웹 애플리케이션 버전 1.18의 기능적, 비기능적 요구사항을 정의합니다.

---

## 1. 버전 1.18 주요 변경 사항 요약

*   **로그인 오류 메시지 개선**: 불명확한 피드백(버퍼링 등)을 제거하고 "닉네임이나 비밀번호가 틀렸습니다"라는 명확하고 통일된 메시지를 제공.
*   **UI 레이아웃 수정**: 데이터 업로드 페이지에서 이미지 미리보기 영역이 좁아져 찌그러지는 문제를 해결하기 위해 미리보기 영역을 버튼 하단으로 분리 및 이동.
*   **프로필 사진 기능 추가**: 내 정보 페이지에서 사용자가 직접 프로필 사진을 업로드하고 로컬에 저장할 수 있는 기능 구현.
*   **문서 관리 강화**: 버전별 README 파일 생성 정책을 확립하고 `docs/versions/` 폴더에 관리.

---

## 2. 상세 기능 요구사항

### 2.1. 사용자 인증 및 피드백 (User Auth & Feedback)
-   **로그인 실패 처리**:
    -   사용자가 입력한 닉네임이 존재하지 않거나, 비밀번호가 일치하지 않는 경우 시스템은 **"닉네임이나 비밀번호가 틀렸습니다."**라는 단일화된 오류 메시지를 즉시 반환해야 한다.
    -   서버 응답 대기 중에는 적절한 로딩 상태를 유지하되, 실패 시 즉시 로딩을 해제하고 경고창(Alert)을 띄워 사용자가 상황을 인지할 수 있게 한다.

### 2.2. 데이터 업로드 UI (Upload UI)
-   **이미지 미리보기 개선**:
    -   '이미지 업로드' 버튼은 순수하게 파일 선택 기능만 수행한다.
    -   선택된 이미지의 썸네일 미리보기는 입력 버튼 그룹(파일, 이미지, 텍스트)의 **하단에 별도의 영역**으로 배치한다.
    -   미리보기 이미지는 1:1 비율을 유지하며 찌그러짐 없이 표시되어야 한다.
    -   각 미리보기 이미지에는 '삭제(X)' 버튼이 있어, 사용자가 잘못 선택한 이미지를 개별적으로 제거할 수 있어야 한다.

### 2.3. 마이페이지 (My Info)
-   **프로필 사진 설정**:
    -   기본 프로필 아이콘을 클릭하여 기기의 갤러리나 파일 탐색기에서 이미지를 선택할 수 있다.
    -   선택한 이미지는 즉시 원형 프로필 영역에 적용된다.
    -   이미지 데이터는 `localStorage`를 활용하여 `spendy_profile_img_{userId}` 키로 저장되어, 재접속 시에도 유지되어야 한다. (백엔드 스키마 변경 없음)

### 2.4. 문서화 (Documentation)
-   버전 업데이트 시마다 `docs/versions/` 경로에 해당 버전의 기능을 명시한 `README_v{버전}.md` 파일을 생성하여 이력 관리를 철저히 한다.

---

## 3. 비기능 요구사항

-   **반응형 디자인**: 이미지 미리보기 그리드와 프로필 사진 업로드 UI는 모바일과 데스크톱 환경 모두에서 자연스럽게 동작해야 한다.
-   **데이터 지속성**: 프로필 사진은 로컬 스토리지 용량 한도 내에서 base64 문자열로 안전하게 저장 및 로드되어야 한다.
