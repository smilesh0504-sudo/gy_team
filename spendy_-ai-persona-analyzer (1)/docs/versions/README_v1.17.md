# Spendy: AI 페르소나 분석기 - 버전 1.17 요구사항 정의서

이 문서는 'Spendy' 웹 애플리케이션 버전 1.17의 기능적, 비기능적 요구사항을 정의합니다. 이 파일은 버전별 변경 사항을 추적하기 위해 생성되었습니다.

---

## 1. 버전 1.17 주요 변경 사항 요약

*   **문서 관리 체계 변경**: 단일 `README.md` 업데이트 방식에서 `docs/versions/` 폴더 내 버전별 파일 생성 방식으로 변경하여 히스토리 추적을 용이하게 함.
*   **회원가입 정책 강화**:
    *   닉네임 길이 13자 제한 도입.
    *   비밀번호 정책 변경 (5~15자 → 4~14자).
*   **사용자 편의성 개선**:
    *   비밀번호 확인 기능 및 실시간 일치 여부 피드백(텍스트 및 색상) 제공.

---

## 2. 상세 기능 요구사항

### 2.1. 사용자 인증 (User Authentication)

#### 2.1.1 회원가입 (Sign Up)
-   **닉네임 제약 조건**:
    -   닉네임은 **최대 13자**까지만 입력 가능하다.
    -   13자를 초과하여 입력할 경우, 입력 필드 하단에 "닉네임은 최대 13자까지 가능합니다."라는 붉은색 경고 문구가 표시되며 가입 버튼이 활성화되지 않아야 한다.
-   **비밀번호 제약 조건**:
    -   비밀번호 길이는 **4자 이상 14자 이하**여야 한다. (기존 5~15자에서 변경)
    -   반드시 **영문자와 숫자가 혼합**되어야 한다.
    -   조건을 만족하지 않을 경우 "4~14자, 영문과 숫자를 조합해주세요."라는 경고 문구가 표시된다.
-   **비밀번호 확인 기능**:
    -   사용자는 비밀번호를 재입력하여 오타를 방지한다.
    -   **일치 시**: 초록색으로 "비밀번호가 일치합니다." 문구가 표시된다.
    -   **불일치 시**: 빨간색으로 "비밀번호가 일치하지 않습니다." 문구가 표시된다.
    -   비밀번호 확인 필드까지 올바르게 입력되어야 가입 버튼이 활성화된다.

#### 2.1.2 로그인 (Login)
-   기존 기능을 유지한다.
-   입력된 비밀번호 정책 변경 사항(4~14자)은 로그인 유효성 검사에도 동일하게 적용되거나, 기존 계정과의 호환성을 고려하여 유연하게 처리된다.

### 2.2. 문서 관리 (Documentation)
-   기능이 추가되거나 변경될 때마다 `docs/versions/` 디렉토리에 `README_v{버전}.md` 형식으로 파일을 생성한다.
-   이를 통해 각 버전에서 어떤 기능이 의도되었는지 명확히 파악할 수 있게 한다.

## 3. 비기능 요구사항

-   **안정성 (Stability)**: 프론트엔드 유효성 검사뿐만 아니라 백엔드(`auth.ts`) 통신 직전에도 데이터를 검증하여 비정상적인 데이터가 DB에 저장되는 것을 방지한다.
-   **코드 품질**: 중복 코드를 제거하고, 조건부 렌더링을 활용하여 UI 코드를 간결하게 유지한다.
-   **사용자 피드백**: 모든 유효성 검사는 사용자가 입력을 마친 후 제출할 때가 아닌, **입력하는 도중(Real-time)**에 피드백을 제공하여 사용성을 높인다.
