
# Spendy: AI 페르소나 분석기 - 버전 1.21 요구사항 정의서

이 문서는 'Spendy' 웹 애플리케이션 버전 1.21의 기능적, 비기능적 요구사항을 정의합니다.

---

## 1. 버전 1.21 주요 변경 사항 요약

*   **경고창(Alert) 시스템 전면 교체**: 브라우저 네이티브 `alert()` 함수가 특정 환경(Dismiss all 체크 등)에서 무시되는 현상을 해결하기 위해 **커스텀 모달(Custom Modal)** 기반의 알림 시스템을 도입.
*   **확실한 오류 피드백 보장**: 로그인 및 회원가입 실패 시, 외부 요인에 의해 차단되지 않는 앱 자체 UI로 오류 메시지를 표시.

---

## 2. 상세 기능 요구사항

### 2.1. 커스텀 알림 시스템 (Custom Alert System)
-   **AlertModal 컴포넌트**:
    -   브라우저의 `window.alert`를 대체하는 React 컴포넌트를 구현한다.
    -   **디자인**: 앱의 테마 컬러(Blue)와 둥근 모서리(Rounded UI)를 적용하여 심미성을 높이고, 배경 흐림 효과(Backdrop Blur)를 주어 사용자 집중도를 유도한다.
    -   **동작**: `App.tsx`의 최상위 레벨에서 렌더링되며, 전역 상태(`alertState`)에 따라 즉시 화면 최상단(z-index 100)에 표시되어야 한다.

### 2.2. 로그인 실패 처리 개선
-   **네이티브 Alert 제거**:
    -   기존 `setTimeout` + `alert()` 로직을 모두 제거한다.
    -   로그인 실패(`success: false`) 시 `showAlert(message)` 함수를 호출하여 커스텀 모달을 띄운다.
-   **사용자 경험**:
    -   "Dismiss all" 등의 브라우저 차단 설정과 무관하게 무조건 메시지가 표시되어야 한다.
    -   확인 버튼을 누르면 모달이 부드럽게 닫혀야 한다.

---

## 3. 비기능 요구사항

-   **안정성**: 브라우저 호환성 및 사용자 설정에 영향을 받지 않는 독립적인 UI 컴포넌트를 사용하여 기능의 신뢰성을 확보한다.
-   **심미성**: 경고창 또한 앱의 일부로서 디자인 일관성을 유지한다.
